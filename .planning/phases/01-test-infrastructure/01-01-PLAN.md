## Threats and Mitigations (ASVS L1)

This is a static personal portfolio (no auth, no server, no user data). Phase 1 only changes build/dev tooling. Threat surface is narrow.

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Supply-chain: `vitest` and `jsdom` pull transitive devDeps that could contain malicious packages | Low | Pin packages via `package-lock.json` after install; review `npm audit` output post-install. devDeps do not ship to production. |
| Test code accidentally embeds real EmailJS credentials (visible in `src/pages/Contact.jsx`) | None — out of scope | Phase 1 does not touch Contact.jsx. Phase 3 will mock EmailJS, not use real keys. |
| `jsdom` environment exposes browser APIs that could be exploited in test code | None | Tests run locally / in CI only, never deployed. |

**No HIGH-severity threats.** Block-on threshold is HIGH; gate passes.
</threat_model>
