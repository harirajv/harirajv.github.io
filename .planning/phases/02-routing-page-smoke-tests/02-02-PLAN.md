Side-effect contracts per page:

**Home** (src/pages/Home.jsx lines 6–19):
- `componentDidMount` calls `this.typed = new Typed(this.el, options)`.
- `componentWillUnmount` calls `this.typed.destroy()`.
- D-05 mock pattern (declared at module scope of Home.test.jsx — Vitest hoists `vi.mock`):
  ```
  vi.mock('typed.js', () => ({
    default: class { constructor() {} destroy() {} }
  }));
  ```
- The ref `this.el = el` is set by React on the `<span ref={...}/>` after the first render — the mocked constructor accepts it and ignores it.

**Portfolio** (src/pages/Portfolio.jsx lines 12–21):
- `componentDidMount` calls `fetch("assets/projects.json").then(r => r.json()).then(data => this.setState({...})).catch(...)`.
- The component's `render()` already returns `<section id="portfolio">` regardless of state — the fetch only fills in additional `projects` markup. Therefore `#portfolio` is in the DOM synchronously after the first render, BUT the in-flight fetch promise will reject (real fetch in jsdom) and noisily log errors unless spied. D-06 mandates the spy.
- D-06 spy pattern (per-test, in `beforeEach`):
  ```
  vi.spyOn(global, 'fetch').mockResolvedValue({
    ok: true,
    json: () => Promise.resolve({ projects: [] })
  });
  ```
- After render, use `await waitFor(() => expect(container.querySelector('#portfolio')).toBeInTheDocument())` so the assertion runs after the fetch chain settles (per `<specifics>` in CONTEXT.md — prevents false-pass timing).
- Restore the spy in `afterEach` via `vi.restoreAllMocks()`.

**Contact** (src/pages/Contact.jsx lines 51–72):
- Imports `@emailjs/browser` at module top but `emailjs.sendForm` only fires on `onSubmit`. D-07 — no mock needed at render time.
- Form field `name` attributes (the contract — confirmed by reading Contact.jsx):
  - `<input type="text" name="user_name" ... />` — line 54
  - `<input type="email" name="user_email" ... />` — line 57
  - `<input type="text" name="subject" ... />` — line 61
  - `<textarea name="message" ... />` — line 64
- Submit button: `<button type="submit">Send Message</button>` — line 71

**About** and **Resume**: no `componentDidMount`, no side effects, no mocks (D-08).

- Do NOT touch any of the page bodies (`Home.jsx`, `About.jsx`, `Resume.jsx`, `Portfolio.jsx`, `Contact.jsx`) — Phase 2 only ADDS tests (CONTEXT.md "Files to leave alone" scope guard).
